Multi-City Journey: Flight suggestions

This decision has the biggest impact on scope. What should the app do between cities?

A

Deep link to Google Flights

App shows a "Fly Tokyo → Singapore" card in the itinerary. Tapping opens Google Flights / Skyscanner pre-filled with cities + dates. No pricing in-app, no API needed.

B

Live flight search (Amadeus / Skyscanner API)

App queries a flight API and shows real prices + departure times inline. High complexity, API cost, requires airline data agreements.

C

Transit block only — no flight UI

The itinerary just has a "Travel day: Tokyo → Singapore" placeholder. Booking is entirely the user's responsibility. Keeps the app focused on the on-ground experience.
