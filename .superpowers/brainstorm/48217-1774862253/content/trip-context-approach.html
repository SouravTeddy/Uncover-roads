Issue 2 — Trip Context Questions

Before generating an itinerary, the app needs a few travel details. Where should these questions live?

A

Bottom sheet, after "Create Itinerary"

When the user taps Create Itinerary on the map, a modal sheet slides up asking 3–4 quick questions (date, starting point, days). They answer and hit Generate. No new screen in the navigation.

B

Dedicated "Trip Setup" screen

A new full-screen step is inserted between the map and the itinerary. The user sees a clean form with travel date, starting point type, arrival time, and number of days. Feels like a natural checkpoint.

C

Inline in the itinerary loading screen

While the itinerary is generating, a small card asks the questions one at a time. Answers refine the result live. Feels minimal but couples form + loading state.

Questions to collect

  • 📅 Travel date
  • 🏨 Starting point — hotel, airport, station, or Airbnb
  • 🕐 Arrival / flight time (if airport/station)
  • 📆 Number of days & which day of the trip
