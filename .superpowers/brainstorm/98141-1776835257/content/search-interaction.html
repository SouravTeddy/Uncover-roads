How should the search input work?

The key UX decision: how does the "controlled" dropdown guide users toward structured queries?

🔍 museum near shinjuku...
🏛 Museum near Shinjuku Station
🏛 Museum near Shinjuku Gyoen
🏛 Museum near Shinjuku (district)

A — Smart Suggestions

As the user types, AI parses partial input in real-time and surfaces structured suggestions. Users pick from the dropdown rather than hitting enter on free text. Works like a smart autocomplete that understands intent.

🔍 What are you looking for?
🗺 Place type near location
🎉 Events on date
⭐ Best of category in this area

B — Template Picker

Tapping the search bar shows pre-built query templates. User picks a template and fills in the blanks (e.g., taps "Events on [date]" → date picker appears). Very controlled, no ambiguity, but less natural to discover.

C — Grouped Results Dropdown

Type anything, get a grouped dropdown that separates "Places" from "Events" from "Areas". Selections are always structured under the hood. The user feels free to type naturally while we control what can actually be selected.
