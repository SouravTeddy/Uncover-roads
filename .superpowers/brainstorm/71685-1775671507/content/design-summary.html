1 · Two chip types

☕ Café nearby ›
🚻 Restrooms ↗

Indigo › — tapping opens an inline list of 2–3 nearby results fetched from Google Places Nearby Search (new /nearby endpoint). User picks one and taps "Go ↗" to open in Maps.

White ↗ — tapping opens Apple Maps (iOS) or Google Maps (Android/web) directly, pre-filled with a search query near the stop's coordinates. No list, no delay.

3 · Data & backend

Chip selection — reads Google types[] from the existing usePlaceDetails cache (already fetched when user tapped the place on the map). No new fetch needed to pick chips.

Expand results — new GET /nearby?lat=&lon=&type=&radius=500&limit=3 endpoint, called only when user taps an expand chip. Returns name, rating, distance, lat/lon. Cost: ~$0.032 per tap.

Direct chips — construct a Maps deep-link client-side using the stop's lat/lon already in ItineraryStop. Zero API cost.

4 · Expand UX

Tapping an indigo chip highlights it and opens a panel below with 3 results (name, walk time, rating). Each has a "Go ↗" button that opens Maps directions. Tapping the chip again collapses it. Only one chip expands at a time.
